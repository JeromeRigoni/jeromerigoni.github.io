Vision Classifier

Classification d'images dans le navigateur · TensorFlow.js · MobileNet v2

Stack TensorFlow.js · MobileNet v2 Où 100% navigateur Modèle ~14 Mo (téléchargé une fois)
TL;DR

Tu glisses une image, MobileNet v2 (CNN pré-entraîné sur 1.4M d'images ImageNet) tourne dans ton navigateur via WebGL, et te renvoie les 5 classes les plus probables avec leur score. Zéro upload, zéro serveur, zéro tracking.

Glisse une image

Photo, screenshot, illustration. Premier chargement plus lent (téléchargement du modèle), ensuite quasi-instantané.

    // 02 — comment ça marche

    MobileNet v2, en 30 secondes

    MobileNet est un CNN conçu par Google (2018) pour tourner efficacement sur mobile. Comparé à un VGG16 traditionnel (~138M paramètres), MobileNet v2 en a seulement ~3.5M, grâce à des convolutions séparables en profondeur.

    Pipeline d'inférence

    1. L'image est redimensionnée en 224×224
    2. Les pixels sont normalisés ([-1, 1])
    3. Forward pass dans le réseau (WebGL)
    4. Softmax → 1000 probabilités
    5. Top-5 sélectionné et affiché

    Pourquoi côté client ?

    • Privacy — aucune image n'est uploadée
    • Latence — pas de round-trip serveur
    • Coût — 0€ d'infra à provisionner
    • Offline-ready — marche après le premier chargement
    // 03 — limites

    Ce que ce modèle ne sait pas faire

    • Labels anglais uniquement — ImageNet n'est pas localisé
    • 1000 classes seulement — et certaines très spécifiques (African vs Indian elephant)
    • Pas de détection multi-objet — répond « qu'est-ce que c'est ? », pas « où est quoi ? »
    • Souffre sur les images hors-distribution — sketches, captures d'écran, memes
    • Premier chargement ~14 Mo — le modèle est mis en cache ensuite

    Pour aller plus loin : fine-tuner sur un dataset custom (Teachable Machine, ou transfer learning manuel), ou passer à un modèle de détection type COCO-SSD pour identifier plusieurs objets dans une même image.
